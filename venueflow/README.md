# VenueFlow AI

A next-generation smart stadium experience platform designed to solve the critical challenges of crowd congestion, long wait times, and poor real-time communication at large-scale sporting venues.

## Project Architecture & Logic Locations
- **Frontend (UI):** `src/app/`
  - Attendee Side: `src/app/dashboard/page.tsx`
  - Admin Side: `src/app/admin/page.tsx`
- **Backend (API):** `src/app/api/`
  - `admin/simulate`: Handles manual state changes.
  - `routing`: Dijkstra-based pathfinding logic.
  - `zones`: Real-time data retrieval.
- **Database Schema:** `prisma/schema.prisma` (SQLite definitions for Zones, WaitTimes, Routes, etc.)
- **Simulation Engine:** `server.mjs` (Integrated into the custom server; handles background crowd fluctuations and status propagation).
- **WebSocket Logic:** `server.mjs` (Server-side broadcast) and `src/hooks/useSocket.ts` (Client-side listener).
- **Routing Logic:** `src/lib/routing.ts` (Implementation of the least crowded path algorithm).

## Setup Instructions
1. **Install dependencies:** `npm install`
2. **Setup Database:** `npx prisma db push`
3. **Seed Dummy Data:** `node prisma/seed.js`
4. **Start Application:** `node server.mjs` (Important: Use this instead of `npm run dev` to enable WebSockets & Simulation)
5. **Open Browser:** [http://localhost:3000](http://localhost:3000)

## Environment Variables (.env example)
Since this project uses SQLite for an autonomous out-of-the-box local demo, the `.env` file generated by Prisma only requires the local file path:
```env
DATABASE_URL="file:./dev.db"
```

## Hackathon Project Summary
**VenueFlow AI** is built with a full-stack architecture (Next.js, Tailwind CSS, Prisma, SQLite). It features:
- **Fan-Facing Dashboard:** A highly polished, mobile-first web app providing attendees with real-time venue heatmaps, queue intelligence (fastest food/restrooms), smart matchday alerts, and routing suggestions based on current congestion.
- **Admin/Operations Portal:** A premium, dark-mode command center for venue staff to monitor live crowd density via interactive visual heatmaps, analyze metrics in real-time, and execute dynamic crowd control measures (e.g., simulating gate lockouts, broadcasting rerouting alerts).

## Sample User Stories
- **As an Attendee**, I want to know which restroom has the shortest line right now, so I don't miss the game while queuing.
- **As an Attendee**, I want to receive push alerts if my assigned entry gate is overcrowded, so I can use an alternate gate and save time.
- **As a Venue Operator**, I want a live thermal heatmap of the entire stadium, so I know exactly where choke points are forming in real-time.

## Sample Admin Workflows
**Workflow: Handling a Halftime Rush**
1. **Monitor:** Admin watches the interactive `Live Crowd Heatmap` as half-time approaches.
2. **Identify:** The system auto-highlights "Gate A" turning red (95%+ density).
3. **Simulate & Alert (Demo):** Admin clicks the "Simulate Gate Overload" in the simulator panel.
4. **Broadcast:** An alert "Gate A Overload. Redirecting to Gate C." is dispatched to all attendees.
5. **Observe Response:** Admin watches as the mock system redistributes the crowd flow, seeing the wait time charts update dynamically while the Fan app adjusts its recommendations.

## API Routes Documentation
- `GET /api/zones`: Fetches all venue zones, current density levels, and trailing wait times.
- `GET /api/alerts`: Fetches the active alerts feed, ordered by timestamp in descending order.
- `POST /api/admin/simulate`: Allows the Admin to inject mock events into the database to trigger real-time updates across the dashboard.
